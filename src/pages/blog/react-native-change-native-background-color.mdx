## Introduction

React Native sets the default background color of the root view to white or to `systemBackgroundColor` if you are running iOS 13+. This is fine for most of the apps, but sometimes you might want to change the color to something else. In this article I will show you how to do it.

?> üí° `systemBackgroundColor` is a new color introduced in iOS 13. It is white in light mode and black in dark mode.

## How to change the background color?

First of all, let's create a new React Native project:

```bash
npx react-native@latest init BackgroundColorExample
```

then `cd` into your project and open the project in Xcode:

```bash
xed ios
```

Now open `AppDelegate.mm` and override following method: 

```objc
- (UIView *)createRootViewWithBridge:(RCTBridge *)bridge moduleName:(NSString *)moduleName initProps:(NSDictionary *)initProps {
  UIView *rootView = [super createRootViewWithBridge:bridge moduleName:moduleName initProps:initProps];
  rootView.backgroundColor = [UIColor redColor];
  return rootView;
}
```

?> üí° `backgroundColor` property expects `UIColor` object. You can use [uicolor.io](https://www.uicolor.io/) to convert hex to `UIColor` initializer.

This looks great in one appearance, but what if you want to change the background color based on the current appearance? Let's say you want to have red background in light mode and blue background in dark mode.

## Changing background color based on the current appearance

If you are targetting iOS 13+ you can use `colorWithDynamicProvider` to achieve this:

```objc
- (UIView *)createRootViewWithBridge:(RCTBridge *)bridge moduleName:(NSString *)moduleName initProps:(NSDictionary *)initProps {
  UIView *rootView = [super createRootViewWithBridge:bridge moduleName:moduleName initProps:initProps];
  rootView.backgroundColor = [UIColor colorWithDynamicProvider:^(UITraitCollection *traitCollection){
    if (traitCollection.userInterfaceStyle == UIUserInterfaceStyleDark) {
      return [UIColor blueColor];
    } else {
      return [UIColor redColor];
    }
  }];
  return rootView;
}
```

Otherwise, you we will create a new class called `AppearanceAwareRootView` that will be responsible for creating the root view and updating the background color based on the current appearance.

Let's start by creating a header file: `AppearanceAwareRootView.h` with following content:

```objc
#import <React/RCTBridge.h>

@interface AppearanceAwareRootView: UIView

- (instancetype)initWithBridge:(RCTBridge *)bridge
                    moduleName:(NSString *)moduleName
             initialProperties:(NSDictionary *)initialProperties
                  enableFabric:(BOOL)enableFabric;

@end
```

It has a single initializer that will be responsible for creating the root view and setting up the constraints.

Now let's create the implementation file: `AppearanceAwareRootView.mm` with following content:

x> ‚ùóÔ∏è Make sure to change the file extension to `.mm` instead of `.m` as `RCTAppSetupDefaultRootView` is an Objective-C++ function.

```objc
#import <RCTAppSetupUtils.h>
#import "AppearanceAwareRootView.h"

@implementation AppearanceAwareRootView

- (instancetype)initWithBridge:(RCTBridge *)bridge moduleName:(NSString *)moduleName initialProperties:(NSDictionary *)initialProperties enableFabric:(BOOL)enableFabric {
  
  if (self = [super init]) {
    UIView *rootView = RCTAppSetupDefaultRootView(bridge, moduleName, initialProperties, enableFabric);
    [self addSubview:rootView];
    [self setupConstraintsForSubview:rootView];
    [self updateBackgroundColor:rootView];
  }
  
  return self;
}

/**
 Setup constraints for `subview` to fit the whole parent
 */
- (void)setupConstraintsForSubview:(UIView *)subview {
  subview.translatesAutoresizingMaskIntoConstraints = NO;
  
  NSLayoutConstraint *leadingConstraint = [subview.leadingAnchor constraintEqualToAnchor:self.leadingAnchor];
  NSLayoutConstraint *trailingConstraint = [subview.trailingAnchor constraintEqualToAnchor:self.trailingAnchor];
  NSLayoutConstraint *topConstraint = [subview.topAnchor constraintEqualToAnchor:self.topAnchor];
  NSLayoutConstraint *bottomConstraint = [subview.bottomAnchor constraintEqualToAnchor:self.bottomAnchor];
  
  [NSLayoutConstraint activateConstraints:@[leadingConstraint, trailingConstraint, topConstraint, bottomConstraint]];
}

/**
Update subview background color based on current `userInterfaceStyle`
 */
- (void) updateBackgroundColor:(UIView *)subview {
  if (self.traitCollection.userInterfaceStyle == UIUserInterfaceStyleDark) {
    subview.backgroundColor = [UIColor blueColor]; // Dark Appearance
  } else {
    subview.backgroundColor = [UIColor redColor]; // White Appearance
  }
}

/**
 Listen to changes of `traitCollection` to update background color
 */
- (void)traitCollectionDidChange:(UITraitCollection *)previousTraitCollection {
  [self updateBackgroundColor:self.subviews.firstObject];
}

@end
```



Now inside `AppDelegate.mm` change the method to following:

```objc
- (UIView *)createRootViewWithBridge:(RCTBridge *)bridge moduleName:(NSString *)moduleName initProps:(NSDictionary *)initProps {
  BOOL enableFabric = NO;
#if RCT_NEW_ARCH_ENABLED
  enableFabric = self.fabricEnabled;
#endif
  return [[AppearanceAwareRootView alloc] initWithBridge:bridge moduleName:moduleName initialProperties:initProps enableFabric:enableFabric];
}
```

The `AppearanceAwareRootView` will now listen to changes of `traitCollection` and update the background color accordingly.
